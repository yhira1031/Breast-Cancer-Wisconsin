# Breast Cancer Wisconsin: 高精度分類モデルの構築と評価指標の実践的理解

## 1. プロジェクト概要
UCI Machine Learning Repositoryの「Breast Cancer Wisconsin (Diagnostic) Dataset」を用い、細胞核のデジタル画像の特徴量から腫瘍が「良性（Benign）」か「悪性（Malignant）」かを判定する分類モデルを構築しました。

本プロジェクトの最大の目的は、単なる正解率（Accuracy）の追求ではなく、医療データにおいて極めて重要な**「悪性の見逃し（偽陰性: False Negative）」を最小化するプロセス**を実践し、不均衡データの適切な処理方法を学ぶことです。

## 2. 実施プロセス

### ① ドメイン知識に基づく特徴量エンジニアリング
物理的な知見に基づき、がん細胞特有の「いびつさ」や「不均一性」を表現する特徴量を新規作成しました。
- **Circularity_mean (円形度)**: 面積と周囲長から算出。真円からどれだけ歪んでいるかを数値化。
- **Area_Worst_Mean_Ratio (乖離率)**: 腫瘍内の最大細胞と平均細胞のサイズ差。
- **Area_Relative_Error (相対的ばらつき)**: 面積の標準誤差を平均で割った変動係数。

### ② 特徴量選択とモデルの最適化（二段構えのアプローチ）
「次元の呪い」を防ぐため、以下の戦略をとりました。
1. **ランダムフォレストによる選別**: 特徴量の重要度（Feature Importances）を算出し、ノイズの少ない上位10個の変数を厳選。
2. **距離ベースモデルへの適用**: 選別・標準化されたクリーンなデータのみを、特徴量の多さに弱い **SVM (サポートベクターマシン)** や **KNN (K近傍法)** に入力し、モデルのポテンシャルを最大化。

### ③ 厳密な評価パイプラインの構築
- **データ漏洩（Data Leakage）の防止**: 学習データとテストデータを分割した後に、学習データのみを用いて交差検証（Stratified K-Fold）やスケーリングを実施。
- **パイプライン化**: `StandardScaler` とモデルを `Pipeline` で結合し、交差検証時の情報漏洩を完全に遮断。

## 3. 分析結果
厳選した10個の特徴量とクラスウェイト調整を用いたSVMが、最も優れた性能を示しました。

| Model | Accuracy (正解率) | Precision (適合率) | **Recall (再現率/感度)** | F1-score |
| :--- | :--- | :--- | :--- | :--- |
| **SVM (Linear)** | **0.9825** | **0.9841** | **0.9688** | **0.9764** |
| Random Forest | 0.9766 | 1.0000 | 0.9375 | 0.9677 |
| KNN (K=5) | 0.9415 | 0.9821 | 0.8594 | 0.9167 |

結果として、SVMにおいてRecall（悪性の発見率）を **約96.9%** まで引き上げることに成功しました。

## 4. 今回学んだ重要事項（Key Takeaways）

1. **評価指標の真の意味と使い分け**
   - 全体の正解率（Accuracy）が高くても、実務では意味をなさない場合がある。
   - 医療診断や人命・重大な損失に関わるタスクでは、**Recall（再現率：実際の正例をどれだけ見落とさず拾えたか）**を最重要指標として設定し、混同行列の「偽陰性（FN）」を極限まで減らす調整が必要である。

2. **不均衡データへの適切なアプローチ**
   - 「良性63:悪性37」程度の軽度な不均衡であれば、SMOTEのような人工的なデータ合成（オーバーサンプリング）は過学習やノイズの原因になるリスクがある。
   - アルゴリズムのペナルティを調整する **クラスウェイト（`class_weight='balanced'`）** の適用だけで、安全かつ十分に高い精度を引き出せることを確認した（SMOTEは「正常99.9:異常0.1」のような極度の不均衡でこそ真価を発揮する）。

3. **モデル特性の理解と特徴量選択（Feature Selection）**
   - ランダムフォレスト（木系モデル）はノイズに強いが、SVMやKNN（距離系モデル）は不要な特徴量が多いと精度が急激に低下する（次元の呪い）。
   - 「木系モデルで重要な特徴量をあぶり出し、距離系モデルに綺麗なデータとして渡す」というプロセスが、汎化性能の向上に極めて有効である。

4. **データ漏洩（Data Leakage）の絶対的防止**
   - データをTrain/Testに分割する前に標準化（Scaling）やデータ合成（SMOTE）を行うと、未来の情報が学習に漏れてしまい、未知のデータへの対応力が正しく測れなくなる。評価は常に厳密に切り離された環境で行う必要がある。
